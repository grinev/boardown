## Improve cli commands output

---
id: BD-56
type: feature
status: done
epic: cli
order: 1100
---

Reshape the CLI output around how the board is actually read: three views, then one task.

- The three UI tabs become three commands: backlog, release current, archive; the catch-all board command is removed.
- Listing commands return a compact task summary (id, title, type, status, epic, checklist done/total, notes count) instead of whole task objects; task get stays the full-detail drill-down.
- A single --full flag deepens any listing command by one level.
- Mutating commands acknowledge with an identifier only.
- The command key is dropped from the JSON envelope.
